Ping 

 

    The Ping 'command' is implemented using an ICMP (Internet Control Message Protocol) Echo (Type 8) request sent to a particular destination. If an IP implementation is active on the destination machine, an Echo Reply (Type 0) should be sent back to the source of the message. If the machine is not available, we may also receive a Destination Unreachable (3) or simply time out. In this way we can discover if a remote address is available either for debugging or before embarking on a connect operation.

  • You can save the server's response to a file by selecting enable log check box and entering the full path name of the log file.
  • You can control the time interval between each packet sent to the remote machine by changing the Interval value.
  •  The process of waiting for a response from the remote may take longer period of time you can specify the maximum time that a response is considered Timed-Out by changing the time out value.
  • You can save all  responses to a file by selecting enable log check box and entering the full path name of the log file.

Trace Route

Allows you to view the name and or the IP address of each Hop that a signal must visit in order to travel from your machine to the host machine or vice versa using .

 

A series of ICMP Echo packets is sent out with the address of the ultimate target, with Time To Live (TTL) fields incrementing with each request.

The Time To Live field of the IP header determines how many gateways the message can traverse before it will be tagged as 'TTL exceeded'. As the packet passes through a node, the TTL field is decremented. If the TTL gets to 0, the machine will send back an ICMP reply of 'Time Exceeded', along with at least 8 bytes of the original packet (enough for us to examine some of our original ICMP data and ensure it is a response for us).

If the request reaches the target we receive an Echo reply.

We may also receive a Destination Unreachable message or simply timeout on one or more of the remaining requests in the series.

As we send the series of packets, we start with a TTL of 0. After we receive (or timeout) on this, we increment the TTL and send again. By examining the IP from address of the ICMP responses, we can determine the outgoing route that the packets are taking to the target more or less reliably.

 

 

DNS (Domain Name Service)

DNS provides you with the means of identifying the IP address of a given domain and other servers associated with this domain name.

 

 

Having to memorize all of IP addresses used around the world would be an intimidating task. To overcome this obstacle, network administrators created files that mapped the IP addresses of hosts into friendly names and referred to these tables whenever they needed. With the introduction of the Internet, this approach gave birth to a much larger database using the Domain Name Service (DNS).

  • Represented by the DNS demo.
  • The DNS entries returned contain the following information:
    Host  - The domain name to which this resource entry pertains
    Type - A (address), NS (name server), MD (destination), MF (mail forwarder), CNAME (canonical name), SOA (start of authority), MB (mailbox domain), MG (mail group), MR (mail rename domain), NULL (null), WKS (well known service description), PTR (domain name pointer), HINFO (host info), MINFO (mailbox or mail list info), MX (mail server), TXT (text strings)
    lTTL - (Time to Live) a 32 bit signed integer that specifies the time interval that the resource record may be cached before the source of information should again be consulted. 0 value means that the result should not be cached.
    Data and Data2 - Name servers manage two kinds of data. The first kind of data held in sets called zones; each zone is the complete database for a particular subtree of the domain space. This data is called authoritative. A name server periodically checks to make sure that its zones are up to date, and if not obtains a new copy of updated zones from master files stored locally or in another name server. The second kind of data is cached data which was acquired by a local resolver. This data may be incomplete but improves the performance of the retrieval process when non-local data is repeatedly accessed. Cached data is eventually discarded by a timeout mechanism.
    MX Preference - 16 bit integer which specifies the preference given to this entry among others at the same host. (Mail Exchange Type entries only)
    SOA Serial - (Start of Authority Zone Serial) version number of the original copy of the zone. Zone transfer preserve this value. This value warps should be compared using sequence space arithmetic. (SOA Type entries only)
    SOA Refresh - the time interval before the zone should be refreshed.
    SOA Retry - The time interval that should elapse before a failed refresh should be tried.
    SOA Expire - The upper limit on the time interval that can elapse before the zone is no longer authoritative.
    SOA Minimum - minimum TTL field that should be exported with any entry from this zone
  • Ideal for verifying a domain name.
  • Supports TCP and UDP query format.
  • Complies with RFC 1035.
